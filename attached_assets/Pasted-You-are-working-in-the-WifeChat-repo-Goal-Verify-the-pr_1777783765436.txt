Please verify these priority recommendations against the current repo:

1. API perimeter hardening
- CORS should be allowlist-based via ALLOWED_ORIGINS.
- Production should not use open cors().
- API responses should include X-Request-Id.
- A global JSON error handler should exist.
- express.urlencoded should be removed or limited if unused.
- helmet/security headers may be useful.

2. Cost and timeout guardrails
- max_completion_tokens should be reduced from the current value if it is overly high.
- OpenAI calls should have a request timeout.
- OpenAI client should not need to be recreated per request if a singleton is simpler/safer.
- Consider a temporary env kill switch, e.g. COACH_API_DISABLED=true.
- Consider a temporary process-level daily request cap only if it is simple and clearly marked as non-production.

3. Privacy-safe logging
- Do not log raw user input.
- Do not log generated relationship text.
- Do not log raw model output.
- Do not log full provider error objects if they may include prompts or user content.
- Logs should include only safe metadata: requestId, route/tool, status, error class/status, duration, safety branch, rate-limit/quota events.

4. Deterministic safety intercept
- Add a pre-model tripwire for obvious self-harm, violence, fear for safety, threats, sexual coercion, stalking, or control language.
- If tripped, return a static safety-oriented response and do not call OpenAI.
- This should be a fail-safe tripwire, not a claim of perfect classification.
- Log only safe metadata like event type/category/requestId.
- Confirm whether the frontend can render a safety response shape cleanly.

5. Frontend trust/privacy copy
- Replace any overly broad “Not stored” promise unless it is fully true across app logs, server, provider/proxy, and retention.
- Add a simple /privacy page or equivalent.
- Explain what is sent to the AI provider.
- Explain that the app is not therapy and not emergency support.
- Keep the language truthful and user-friendly.

6. Basic tests
- Add tests later for happy paths, invalid input, passcode, rate limits, malformed model output, safety intercept, CORS allow/deny, and log redaction.

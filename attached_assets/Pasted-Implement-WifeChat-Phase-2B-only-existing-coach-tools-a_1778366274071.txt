Goal:
Allow a user to launch existing supported coach tools from a Loop detail screen and save the resulting local conversation/artifact back into that Loop.

Scope:
1. Update `app/loop/[id].tsx` to show Loop action cards for:
   - before-send
   - repair
   - checkin

2. Each action card should route to the existing coach screen with a `loopId` query param:
   - `/coach/before-send?loopId=<id>`
   - `/coach/repair?loopId=<id>`
   - `/coach/checkin?loopId=<id>`

3. Update `app/coach/[tool].tsx` to support two persistence modes:
   - No `loopId`: preserve existing per-tool message storage exactly as today.
   - With `loopId`: load messages from that Loop’s `messages` filtered by `sourceTool`.

4. In Loop mode:
   - Use the same existing `sendCoach(toolKey, text)` call.
   - Do not change `lib/coach.ts`.
   - Do not send Loop context or profile context to the API.
   - On successful assistant reply:
     - append user and assistant `LoopMessage` records to the Loop
     - set `sourceTool` on both records
     - append a `GeneratedArtifact` with:
       - id
       - loopId
       - sourceTool
       - createdAt
       - title
       - payload: `{ text: replyText }`
       - saved: true
   - On failed send:
     - keep existing visible error behavior
     - do not persist Loop messages or artifacts

5. In Loop mode UI:
   - show a small subtitle/badge that the coach is working inside the selected Loop
   - include a back path to the Loop detail
   - avoid generic chat framing
   - keep “final sending is manual” assumptions intact

6. Keep coming-soon tools disabled:
   - planner stays disabled on mobile
   - practice stays disabled
   - Reality Check remains Phase 3
